# 5. Menus and navigation

The theme is aggressively **menu-driven**. Every clickable navigation
surface — top bar, header nav, homepage cards, hero CTA buttons, footer
columns — reads from a named menu in Hugo's `Site.Menus` API. This means
you reshape the site's information architecture from
`config/_default/menus.<LANG>.toml` (or the `[menu]` blocks of
`hugo.toml`), not by editing templates.

## The full menu inventory

| Identifier         | Used where                                                                | Suggested entries |
| ------------------ | ------------------------------------------------------------------------- | ----------------- |
| `main`             | Top header nav. Supports nested dropdowns (`.HasChildren`).               | 4–7               |
| `top_bar`          | Skinny strip above the header. Hidden if empty AND no `[params.contact]`. | 1–4               |
| `footer_useful`    | Left footer column ("Useful Links")                                       | 3–6               |
| `footer_services`  | Middle footer column ("Related Services")                                 | 3–6               |
| `footer_bottom`    | Inline links beside the copyright row                                     | 2–3               |
| `quick_links`      | Homepage Quick Links grid                                                 | 3–8               |
| `homepage_services`| Homepage Services 4-card section                                          | 4 (others render fine, but 4 is the design intent) |
| `hero_buttons`     | Hero CTA buttons                                                          | 2 recommended (primary + secondary) |

Where a menu is empty, the partial that consumes it suppresses itself.
A site with no `quick_links` menu has no Quick Links section.

## Per-entry params the theme reads

Every menu entry can carry `params` that the theme partials inspect:

```toml
[[homepage_services]]
  name    = "Research"
  weight  = 30
  url     = "/research/"
  [homepage_services.params]
    icon        = "research"             # symbol id in layouts/partials/icons.html
    description = "Research areas and labs"
    external    = false                  # opens in a new tab + adds rel="noopener"
    style       = "primary"              # hero_buttons only: "primary"|"secondary"|"accent"
```

### `icon`

Names a symbol in `layouts/partials/icons.html`. Set without the `icon-`
prefix in the menu config (the template prepends it). Full list:

```
arrow-right   award          book          briefcase     building
calendar      chart          chevron-down  chevrons-left chevrons-right
close         document       email         external      facebook
fax           globe          graduation    image         instagram
library       location       menu          news          phone
play          research       search        twitter       users
youtube
```

Add new icons by appending a `<symbol id="icon-NAME">` to
`layouts/partials/icons.html`.

### `external`

When `true`, the rendered link gets `target="_blank" rel="noopener"`. The
theme also uses this flag in the top bar, header, footer columns and
quick links — anywhere a link is rendered.

### `description`

Used by the `homepage_services` partial as the card body text.

### `style`

Only inspected by the `hero_buttons` partial. Values: `primary`,
`secondary`, `accent`. Defaults to `primary` when omitted. Determines the
button class (`.btn-primary`, etc.).

## Multilingual menus

Define menus per-language using the file convention
`config/_default/menus.<LANG>.toml`. Example layout:

```
config/
└── _default/
    ├── menus.en.toml
    └── menus.ar.toml
```

The language switcher (`layouts/partials/lang-switcher.html`) automatically
links to the matching translation if one exists, else falls back to the
target language's home page. No menu duplication needed for the switcher —
it's generated by iterating `hugo.Sites`.

## Nested menus

Only the `main` menu has dropdown support. Use Hugo's `parent`/`identifier`
pattern:

```toml
[[main]]
  name       = "About"
  identifier = "about"
  weight     = 20

[[main]]
  name   = "Mission"
  parent = "about"
  weight = 21
  url    = "/about/mission/"
```

A parent entry without `url` is rendered as a non-clickable `<span class="nav-label">`
that opens the dropdown — desktop on hover, mobile on tap (see
`assets/js/main.js`).

## Active link highlighting

The header marks the current page with `aria-current="page"` using
`{{ $.IsMenuCurrent "main" . }}`. Styling for this lives in
`assets/css/_header.css`.

## Examples in the wild

`exampleSite/config/_default/menus.en.toml` is the authoritative example —
it exercises every menu identifier the theme reads. Use it as a template
when bootstrapping a new site.
